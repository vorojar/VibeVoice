# 有声书工具 — 项目分析

## 竞争力

| 维度 | 评级 | 说明 |
|------|------|------|
| 逐句精细控制 | **强** | 可逐句编辑文本、设情感、重新生成、撤销，这是大多数 TTS 工具不具备的精细度 |
| 逐句情感指令 | **强** | 不同句子可设不同情感，有声书中对白/旁白语气不同，这个能力很关键 |
| 多角色混排 | **强** | 逐句选择不同声音（预设/声音库），旁白+多角色在同一段落交替，支持混合生成 |
| 分句预览 | **强** | 生成前可预览分句结果，编辑文本、调整情感、插入删除句子，确认无误再生成，节省算力和时间 |
| 语音克隆 | **强** | 上传参考音频即可克隆声音，可快速创建角色声线 |
| 声音设计 | **强** | 用自然语言描述声音特征即可创建新声音并保存到声音库，无需参考音频 |
| 声音库 | **强** | 克隆和设计的声音可保存复用，适合多角色项目 |
| 即时试听 | **强** | 单句试听、句间停顿调节，所听即所得 |
| 多语言支持 | **强** | 10 种语言，覆盖主流有声书市场 |
| 生成历史 | **中** | 左侧栏历史记录列表，可回顾之前的生成结果 |
| 零安装使用 | **中** | Web 单页应用，但依赖本地 GPU 部署，非 SaaS |
| 会话恢复 | **中** | IndexedDB 持久化，刷新不丢失，段落结构在编辑过程中保留 |

## 不足

| 维度 | 评级 | 说明 |
|------|------|------|
| 章节/项目管理 | **缺失** | 没有项目概念，无法管理长篇小说的多章节，只有一个会话 |
| 长文本导入 | **缺失** | 只能手动粘贴到 textarea，无法导入 TXT/EPUB/Word 等文件 |
| 导出格式 | **一般** | 支持 WAV/MP3 导出及 SRT/VTT 字幕导出，但缺少 M4A 格式、元数据编辑、封面嵌入等有声书平台常用功能 |
| 语速/音调控制 | **缺失** | 无法逐句调整语速和音调，只有情感指令间接影响 |
| 发音词典 | **缺失** | 人名、地名、专业术语发音无法自定义纠正 |
| 音频后处理 | **缺失** | 无降噪、均衡、响度标准化，有声书平台对音频参数有严格要求（如 ACX） |
| 背景音乐混合 | **缺失** | 无法叠加配乐或环境音效 |
| 多项目存储 | **缺失** | 只能存一个会话，切换项目会丢失当前进度（历史记录部分缓解） |

## 总结

逐句编辑 + 逐句情感 + 多角色混排是核心差异化优势，市面上很少有工具做到这个粒度。最大短板是**章节/项目管理**和**长文本导入**——这两个是有声书制作的刚需，补上后产品完整度会有质的提升。
